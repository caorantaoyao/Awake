2. 整体架构方案
系统架构设计采用“轻量化本地端 + 飞书生态引擎”模式，最大程度利用飞书原生的低代码/无代码能力（Aily、多维表格、工作流）来承载核心业务逻辑，而本地 React 与 FastAPI 则作为用户触达与第三方服务（邮件）的桥梁。

2.1 架构分层设计
本地应用层 (Local Stack)
- 前端 (React): 负责用户注册交互与状态展示。使用本地 npm 启动，适配快速原型开发。
- 后端 (FastAPI): 充当“粘合剂”角色。处理 SMTP 邮件发送、Websocket/Webhook 回调路由，以及与飞书 OpenAPI 的鉴权交互。
- 服务 (QQ SMTP): 稳定的邮件下发通道，确保 Aily 入口直达学生。
飞书生态层 (Feishu Cloud)
- 逻辑层 (Aily): 承载苏格拉底对话算法，通过 Aily 技能连接多维表格。
- 数据层 (Bitable): 任务、学生信息、打卡记录的单一事实来源（SSOT）。
- 自动化层 (Workflows): 4 条核心流处理消息卡片下发、定时提醒与状态反哺。
2.2 核心数据流转逻辑
1. 用户进入流: 官网注册 (POST) -> FastAPI -> Webhook -> 多维表格记录。
2. Aily 对话流: 学生点邮件链接 -> Aily 小海提问 (AI) -> 识别兴趣/目标 (Tool Call) -> 写入任务表。
3. 行动反馈流: 任务表变动 -> 触发工作流发送飞书卡片 -> 学生点击卡片 -> FastAPI 接收打卡信号 -> 调用 Webhook 更新状态 -> 触发 Aily 后续反馈。
2.3 设计亮点：Advice to Action
该架构通过“飞书卡片”将 Aily 生成的“建议”直接转化为“可打卡任务”，有效解决传统 AI 建议流于表面、无法追踪的问题，实现了完整的闭环逻辑。
MVP 核心目标：
- 学生端闭环：完成从用户注册、AI 苏格拉底式对话引导、微任务分发、提醒打卡到反馈激励的完整闭环。
- 低成本验证：利用飞书 Aily 与工作流作为核心逻辑引擎，降低开发复杂度，快速验证 AAG 模型在学生群体中的有效性。

核心流程说明：
1. 注册链路：用户在 React 前端提交信息，FastAPI 后端通过 QQ SMTP 发送欢迎邮件（含 Aily 入口），同时触发工作流写入多维表格。
2. 对话链路：学生通过邮件链接进入飞书 Aily "小海"，进行苏格拉底式深度对话，生成定制化微行动任务。
3. 行动链路：任务写入多维表格后，由飞书工作流进行定时提醒。学生点击卡片完成打卡，触发后端更新状态并引导至下一轮 AI 反馈。

3. 各模块详细实现说明
3.1 前端页面 (React)
- 注册页：包含姓名、邮箱、年级选择器。提交表单时调用后端 /api/register 接口。
- 环境要求：本地运行，通过 npm run dev 启动，使用 Axios 处理跨域请求。
3.2 后端接口 (Python FastAPI)
- 注册接口 (/api/register)：
  - 调用 smtplib 通过 QQ 邮箱 SMTP 发送 HTML 模版邮件。
  - 发送 POST 请求至飞书工作流 Webhook 地址，传输用户信息。
- 打卡回调接口 (/api/checkin)：
  - 接收前端或工作流的打卡信号。
  - 触发飞书 Webhook 更新多维表格中的任务状态。
- 本地环境：本地启动端口 8000。
3.3 飞书 Aily 智能伙伴配置
- 角色定位：名为“小海”，温和、专业、善于提问。
- 对话逻辑：前 3-5 轮不直接给建议，通过提问引导学生发现兴趣。最后总结建议并生成 1 个具体可执行的微行动。
3.4 飞书多维表格 (Bitable)
- 作为核心数据库：存储用户信息、任务清单、打卡记录。
- 状态同步：利用“自动编号”与“最后更新时间”辅助工作流逻辑。

5. 飞书工作流完整配置说明
本方案共涉及 4 条核心工作流，确保自动化运转：
1. 工作流 A：新用户同步 (Webhook 触发)
  - 触发器：Webhook (接收 FastAPI 注册数据)。
  - 动作：在“学生信息表”中新增记录。
2. 工作流 B：任务下发 (多维表格新增记录触发)
  - 触发器：任务表有新记录。
  - 动作：向指定学生发送飞书消息卡片（含“完成打卡”按钮）。
3. 工作流 C：每日打卡提醒 (定时触发)
  - 触发器：每天 20:00 定时执行。
  - 逻辑：筛选“任务状态 = 进行中”的记录。
  - 动作：向学生发送催促提醒消息。
4. 工作流 D：状态更新与反馈引导 (Webhook 触发)
  - 触发器：学生点击卡片或后端接口调用 Webhook。
  - 动作：更新“任务状态”为“已完成”，并向 Aily 发送信号引导 AI 进行结项对话。

6. 飞书 Aily 配置细节
6.1 System Prompt 设计原则
- 苏格拉底提问法：禁止在对话前 3 轮给出结论。使用“你觉得...”、“这背后带给你的感受是...”等引导性词汇。
- 任务具体化：最后生成的任务必须符合 SMART 原则，且是“微行动”（如：在 B 站观看 5 分钟相关职业视频）。
6.2 知识库与工具
- 知识库：导入《中国中学生兴趣量表》描述、各专业分类简介等 PDF。
- 工具调用：配置“写入多维表格”工具，在对话结束时自动将微行动写入任务表。

8. 关键风险点与注意事项
技术注意事项：
- SMTP 安全：QQ 邮箱需要使用授权码而非密码，注意在 .env 中保护密钥。
- 跨域问题：FastAPI 后端需正确配置 CORS 中间件，否则 React 本地请求会失败。
- Aily 链接时效：发送的 Aily 入口链接应保持稳定，建议使用固定的智能伙伴发布地址。
- 数据一致性：在高并发场景下，工作流写入多维表格可能存在延迟，建议后端在调用 Webhook 后添加适当重试机制。
