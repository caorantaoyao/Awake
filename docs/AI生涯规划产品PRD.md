## 1. 产品概述

### 一句话定义

> **AI Growth Agent = 一个持续理解学生、推动学生行动、连接成长资源、记录成长轨迹，并联合老师与家长共同陪伴成长的智能成长伙伴。**

### 产品愿景

传统的生涯规划产品在做完一次测评或输出一份咨询报告后就结束服务。AI Growth Agent 将学生每一次行动所产生的新数据重新纳入动态成长画像，并基于最新画像持续生成下一阶段的成长任务和资源推荐，使"认识自己 → 开始行动 → 连接资源 → 记录成长 → 再次认识自己"形成不断循环、自我优化的**成长飞轮**。

### 与传统产品的本质区别

| 维度 | 传统产品 | AI Growth Agent |
|------|----------|-----------------|
| 评估方式 | 静态测评 → PDF 报告 → 结束 | AI 聊天 → 持续观察 → 动态画像 → 不断加深理解 |
| 行动转化 | 给建议 → 没有然后 | AI 建议 → Today's Mission → AI Focus → 完成 → 成长值 → 下一步 |
| 资源匹配 | 按专业/热门推荐 | 综合兴趣+能力+成长阶段+近期行为 → 个性化排序 |
| 成长可见性 | 看不到成长，不知道是否有效 | 成长时间轴 → 动态画像 → AI 每周成长报告 |
| 协同方式 | 单用户 | 学生 + 家长 + 导师 + 学校 四方协同 |

## 2. 产品定位

### 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| **学生**（Primary） | 12–18 岁初高中生 | 不知道自己适合什么；知道但不行动；资源太多无法选择；看不到成长 |
| **家长** | 学生家长 | 不了解孩子成长过程；想参与但不知如何支持 |
| **导师/教师** | 班主任、辅导员、生涯导师 | 不了解学生近期变化；辅导缺乏数据支撑 |
| **学校/合作伙伴** | 学校管理者、教育机构 | 需要宏观数据：AI使用率、活动参与率、成长趋势 |

### 核心价值主张

- **对学生**：一个越来越懂你的 AI 伙伴，帮你从"知道"走到"做到"
- **对家长**：看到真实成长过程，而非分数标签
- **对教师**：AI 自动总结学生变化，精准辅导有据可依
- **对学校**：全校成长数据一目了然，教育决策有数据支撑

## 4. 核心功能模块

### 模块一：🧠 Understand — 认识自己（AI 成长伙伴）

#### 痛点

很多学生并不是没有信息，而是**不知道自己喜欢什么**。传统产品的模式是：

```
做一次霍兰德测评 → 看一份职业报告 → 结束
```

#### 解决方案：从静态测评到动态成长画像

```
AI 聊天 → 持续观察 → 记录成长 → 更新画像 → 越来越了解学生
```

#### 核心功能

| 功能 | 描述 |
|------|------|
| **AI 成长伙伴（Awaken）** | 通过自然对话了解学生，而非正式测评问卷 |
| **持续记忆** | 记住学生每次交流内容，不遗忘历史 |
| **兴趣探索** | 在对话中引导学生发现兴趣方向 |
| **成长故事** | 将零散的探索行为串联成有意义的成长叙事 |
| **Dynamic Profile** | 兴趣、能力、成长阶段、人格、探索记录——全部动态更新 |

#### 关键创新

> **不是 AI 给学生建议，而是 AI 越来越了解学生。**

---

### 模块二：🎯 Action — 开始行动（行动闭环）

> ⚡ **这是整个产品最重要的一章，也是核心竞争力。**

#### 痛点

学生最大的问题不是"不知道"，而是**"知道，但是不会开始"**。

传统产品场景：
> AI 说："建议你了解一下产品经理。"  
> 然后呢？  
> ——没有然后。

#### 解决方案：行动闭环

```
AI 建议 → Today's Mission → AI Focus → 完成 → 成长值 → 下一步
```

#### 核心功能

**功能 1：Today's Mission（今日成长任务）**

AI 根据学生当前画像，自动生成一项可立即执行的小任务：

> 📌 今天：观看《产品经理的一天》15 分钟  
> 👆 点击：开始成长

**功能 2：AI Focus（升级版番茄钟）**

不是传统的 25 分钟计时器，而是 **AI 陪伴式专注**：

- 开始后 AI 实时陪伴："已坚持 8 分钟，很棒！"
- 感知状态："今天状态不好？我帮你把 30 分钟拆成 3 个 10 分钟。"
- **AI 会动态调整任务难度**——普通番茄钟没有的能力。

**功能 3：Micro Action（微行动升级路径）**

```
今天：只看一个视频
明天：参加一个讲座
后天：预约一位导师
→ 一点点升级，而非一步到位
```

#### 关键创新

> **真正解决的是 Action Gap（行动鸿沟）——从"知道应该做什么"到"愿意开始并坚持完成"。**

---

### 模块三：🌍 Explore — 连接资源（智能匹配）

#### 痛点

海量资源分散在不同平台（课程、导师、竞赛、活动、实践），学生找不到适合自己的。

#### 解决方案

> **AI 不是推荐职业，而是推荐资源。**

#### 推荐逻辑

AI 综合以下维度自动排序推荐：

```
兴趣 + 能力 + 成长阶段 + Mission 完成情况
                    ↓
        推荐：课程 / 导师 / 活动 / 企业实践 / 大学实验室 / 校友
```

**示例**：
> 学生最近完成 Python 学习任务  
> → AI 推荐：**机器人竞赛**  
> → 而不是：医学讲座

---

### 模块四：📈 Growth — 记录成长（成长可视化）

> ⭐ **这是第二大创新——很多产品没有这一层。**

#### 痛点

- 学生不知道：自己有没有成长
- 老师不知道：学生最近发生了什么
- 家长不知道：孩子到底在干嘛

#### 核心功能

| 功能 | 描述 |
|------|------|
| **Growth Map（成长地图）** | 可视化成长时间轴：高一 → 第一次探索 → 第一次导师 → 第一次竞赛 → 第一次项目 → 高二… |
| **成长画像** | 兴趣变化 + 能力变化 + 行为变化，AI 每周自动更新 |
| **成长报告** | 不是考试报告，而是成长报告：本周探索★★★★★ 行动★★★★ 坚持★★★★★ → 下周建议：尝试数据分析 |

#### 关键创新

> **AI 不是给建议，而是证明成长。**

---

### 模块五：🤝 Ecosystem — 家校协同（四方陪伴）

#### 家长端

看到的不是"孩子适合什么职业"，而是：

> 📋 本周：完成 3 个成长任务 | 参加 2 个活动 | 导师评价：积极 👍

#### 导师端

AI 自动总结学生变化：

> 📊 最近：产品设计兴趣 ↑ | Python ↓  
> 💡 建议：安排一次交流

导师**不用翻聊天记录**。

#### 学校端

宏观数据看板自动生成：

> 📈 全校 AI 使用率 | 活动参与率 | 成长趋势

## 5. 用户路径

### 阶段一：认识自己（Understand）

学生第一次进入平台时，**不会被要求完成一份冗长的职业测评**，而是通过 AI Growth Agent（小海）发起一段自然的对话。

- 在轻松交流中了解学生当前的成长阶段、兴趣方向、学习体验以及未来困惑
- 结合历史探索记录、课程参与情况和成长行为，逐步建立**可持续更新的动态成长画像**
- AI 不再依赖一次性的静态数据，而是随着学生成长不断加深理解

### 阶段二：开始行动（Action）

当 AI 对学生形成初步理解后，**不会直接输出一份复杂的规划报告**，而是根据学生当前发展阶段自动生成一项可立即执行的 Today's Mission。

- 进入 AI Focus 专注模式后，AI 提供陪伴、鼓励和过程反馈
- 根据实际完成情况动态调整任务难度
- 帮助学生将"知道应该做什么"真正转化为"愿意开始并坚持完成"

### 阶段三：连接资源（Explore）

完成成长任务后，系统实时更新画像，并基于最新画像自动匹配最适合当前阶段的成长资源。

- 推荐范围：课程、校园活动、行业导师、实践项目、竞赛机会
- 与传统按专业或热门程度推荐不同，AI 综合学生近期探索方向、成长轨迹和行为反馈进行推荐
- 真正做到**"因人而异、因时而异"**

### 阶段四：记录成长（Growth）

每次成长任务、活动、导师交流后，系统自动沉淀成长数据，持续更新 Growth Map。

- 学生可看到兴趣和能力的变化轨迹
- 通过成长时间轴回顾每一个关键节点
- AI 定期生成成长报告，规划下一阶段目标
- 成长变得**清晰、连续且可追踪**

### 阶段五：家校协同（Ecosystem）

家长和导师同步接收 AI 自动生成的成长洞察（不是分数和标签）。

- 导师快速了解学生近期探索行为和需要关注的方向 → 提供精准辅导
- 家长清晰看到孩子完成了什么、参与了什么、形成了什么新兴趣
- 在充分尊重学生自主成长的前提下，构建**长期、持续的人机协同成长机制**

## 6. 成长飞轮

> 这五个模块不是五个独立功能，而是一个完整的闭环。

```
                     🧠 Understand
                  持续理解学生，更新画像
                        │
                        ▼
                     🎯 Action
                把建议变成每日可完成的行动
                        │

```

## 7. 技术架构

### 7.1 技术栈分层

```
┌─────────────────────────────────────────────┐
│              Application Layer               │
│  Today's Mission │ Growth Map │ Explore │ Report │
├─────────────────────────────────────────────┤
│              Workflow Layer                  │
│         飞书 Workflow（Agent 编排）           │
├─────────────────────────────────────────────┤
│              Knowledge Layer                 │
│       飞书知识库（RAG，降低幻觉）              │
├─────────────────────────────────────────────┤
│                AI Layer                      │
│     豆包 / 飞书 Aily（可替换 DeepSeek、Qwen）  │
├─────────────────────────────────────────────┤
│               Data Layer                     │
│  飞书多维表格（学生画像、成长记录、资源库）     │
└─────────────────────────────────────────────┘
```

### 7.2 四大核心技术

#### ① RAG（Retrieval-Augmented Generation）

**解决**：AI 幻觉问题。

> 学生问："生物医学工程就业怎么样？"  
> → AI 不是自由回答  
> → 知识库检索 → 基于可信知识总结

#### ② Long-term Memory（长期记忆）

**解决**：AI 不会忘记学生。

> 三个月前：学生喜欢建筑  
> 今天：喜欢工业设计  
> → AI 发现兴趣变化 → 自动更新画像

#### ③ Agent Workflow（智能体工作流）

**解决**：AI 不仅能聊天，还能真正工作。

> 推荐任务 → 创建日历 → 发送提醒 → 记录成长 → 更新画像 → 生成报告  
> → Agent 自主完成全链路

#### ④ Recommendation Engine（推荐引擎）

**解决**：资源精准匹配。

> 输入：兴趣 + 能力 + 成长阶段 + 完成情况  
> 输出：个性化排序的课程 / 导师 / 实践资源

## 8. 关键约束与风险

### 8.1 平台约束

本方案优先基于**飞书现有能力**实现，遵循 **"配置优先、开发最少"** 原则：

- 飞书知识库、Aily、Workflow、多维表格、日历及消息能力
- MVP 阶段不涉及自研大模型训练
- 不开发复杂推荐算法
- 利用飞书 AI Agent 编排能力快速验证核心价值

> **我们不是在空想一个产品，而是在飞书生态里设计一个真正能够落地的 AI Agent。**

### 8.2 数据资源限制 ⚠️

| 风险 | 应对 |
|------|------|
| 目前没有真实学生成长数据 | 成长画像采用模拟数据，后续接入学校真实数据 |
| 课程/导师/活动/实践数量有限 | MVP 先接入海亮内部资源，后续开放第三方 |

### 8.3 AI 伦理限制

> **AI 不能决定学生未来，只能提供建议。最终由老师、学生、家长共同决定。**  
> 这是教育伦理底线。

### 8.4 开发资源约束

| 条件 | 应对策略 |
|------|----------|
| 比赛开发周期短 | MVP 只验证 Action 闭环，不做复杂推荐算法 |
| 人力：1 前端 + 1 算法 + 1 产品 | 优先使用飞书低代码、Workflow、知识库，减少定制开发 |

### Phase 1 — MVP（2–4 周）

**目标**：可跑通 Demo，验证 Action 闭环

### A. 关键术语表

| 术语 | 定义 |
|------|------|
| **AI Growth Agent（Awaken）** | 产品核心 AI 智能体，承担理解、推动、推荐、记录和协同五大职责 |
| **Dynamic Profile** | 动态成长画像，取代传统静态测评报告 |
| **Today's Mission** | AI 每日生成的个性化成长小任务 |
| **AI Focus** | AI 陪伴式专注模式，可动态调整任务难度 |
| **Micro Action** | 微行动升级路径，从简单任务逐步升级 |
| **Growth Map** | 可视化成长时间轴 |
| **Action Gap** | 从"知道"到"做到"之间的鸿沟——本产品的核心攻克目标 |
| **成长飞轮** | Understand → Action → Explore → Growth → Ecosystem 的闭环循环 |

### B. 竞品差异化总结

| 对比维度 | 传统生涯规划产品 | AI Growth Agent |
|----------|-----------------|-----------------|
| 评估 | 一次性静态测评 | 持续动态画像 |
| 建议 | 输出报告即结束 | 任务 → 行动 → 反馈 → 再推荐 |
| 资源 | 按分类浏览 | 个性化 AI 排序推荐 |
| 成长 | 无追踪 | 时间轴 + 每周报告 |
| 协同 | 单人使用 | 学生+家长+导师+学校四方 |
| 技术底座 | 自研或第三方 | 飞书生态原生能力 |
