# Conversation Recorder - 对话全链路记录器

将当前 AI 会话的完整上下文结构化保存到磁盘，便于后续复盘、审计、知识沉淀与问题追溯。

## 何时调用

**必须主动调用此 Skill 当：**
1. 用户明确说："保存对话"、"记录会话"、"存档"、"保存这次交流"、"把这次对话记下来"
2. 用户说："沉淀这次对话"、"记住今天的讨论"
3. 完成一个重要功能开发、Bug 修复、架构决策后
4. 会话即将结束前用户要求总结归档

**不要在以下情况调用：**
- 简单的问答对话（单次交互即完成）
- 用户明确表示不需要记录
- 敏感信息处理场景（密码、密钥等）

## 存储位置与命名规范

所有记录保存在项目根目录下：
```
.trae/conversations/
└── YYYYMMDD/
    └── {session-slug}-{timestamp}.md
```

- `YYYYMMDD`：当前日期文件夹
- `{session-slug}`：从用户初始目标提炼的简短英文标识（如 `onboarding-flow-implementation`、`auth-bug-fix`），全小写，连字符分隔
- `{timestamp}`：Unix 时间戳后 6 位，避免重名

## 记录结构（Markdown 格式）

每份记录必须包含以下章节，严格按顺序组织：


## 执行步骤

调用此 Skill 后，按以下步骤执行：

### Step 1: 收集会话上下文
1. 读取当前会话的所有对话历史，从第一轮开始
2. 提取 `chat_session_id` 和时间戳信息
3. 识别用户初始目标和最终达成状态

### Step 2: 生成 slug 与路径
1. 从用户初始目标中提炼 3-6 个英文关键词作为 slug
2. 创建日期文件夹（如不存在）
3. 生成最终文件路径

### Step 3: 填充记录模板
1. 按上述结构逐轮填写
2. **工具调用与文件操作必须如实记录**，不得省略
3. **AI 思考过程**要诚实记录关键判断，不要事后美化
4. **代码变更**需标注具体文件路径和变更内容

### Step 4: 写入并验证
1. 使用 Write 工具将完整记录写入目标路径
2. 读取文件确认写入成功
3. 向用户反馈：
   - 记录保存路径
   - 记录涵盖的轮次数
   - 是否有未完整捕捉的内容需要补充

## 质量要求

- **完整性**: 不得遗漏任何一轮用户输入和工具调用
- **真实性**: AI 思考过程如实记录，包括错误的尝试和被纠正的过程
- **可追溯性**: 所有提到的代码文件必须使用可点击的 `[filename](file:///absolute/path)` 格式
- **时间准确性**: 时间戳使用真实的工具调用和消息时间，不要编造

## 输出格式约束

- 所有文件路径使用绝对路径
- Markdown 表格对齐整齐
- 代码块标注正确的语言
- 中文内容使用中文标点
- 长内容适当折叠或分段，保证可读性

## 示例触发场景

```
用户: "帮我实现登录功能，用 JWT"
...（多轮对话后完成功能）...
用户: "好的，保存这次对话"

→ 此时调用 conversation-recorder，将整个登录功能实现过程存档
```

```
用户: "这个 Bug 终于修好了，记录一下整个排查过程"

→ 调用 conversation-recorder，重点记录排查思路、尝试过的错误路径、最终根因
```
